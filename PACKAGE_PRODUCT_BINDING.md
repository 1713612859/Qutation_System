# 套餐产品绑定功能说明

## 功能概述

套餐可以绑定多个产品。在新增/编辑套餐表单时，可以直接添加产品列表，保存时会自动保存所有产品关联。

## 后端逻辑

### 1. PackageEntity（套餐实体）
- 添加了 `items` 字段（`List<PackageItem>`），非数据库字段，用于业务逻辑处理
- 保存时可以通过 `items` 字段一次性保存所有产品

### 2. PackageServiceImpl（套餐服务）
- **save() 方法**：
  - 新增时：先保存套餐基本信息，获得ID后保存产品列表
  - 更新时：先删除旧的套餐明细，再保存新的产品列表
  - 保存产品时会自动填充产品名称和SKU
  - 使用事务确保数据一致性

- **findById() 方法**：
  - 自动加载套餐的产品列表
  - 填充产品信息（产品名称、SKU）

- **delete() 方法**：
  - 级联删除套餐明细

### 3. PackageController（套餐控制器）
- **POST /api/packages**：保存套餐（包含items列表）
- **PUT /api/packages/{id}**：更新套餐（包含items列表）
- **GET /api/packages/{id}**：获取套餐详情（包含items列表）

## 前端逻辑

### 1. PackageList.vue（套餐管理页面）

#### 新增/编辑表单功能：
- **基本信息**：套餐代码、名称、货币、折扣、税率、描述
- **产品列表管理**：
  - 添加产品：点击"添加产品"按钮，可以在表格中新增一行
  - 选择产品：下拉选择产品，自动填充产品名称和SKU
  - 编辑产品信息：数量、单价、折扣、税率、备注
  - 删除产品：删除表格中的产品行
  - 保存时：产品列表会随套餐一起保存到后端

#### 产品列表表格字段：
- 产品（下拉选择）
- 数量
- 单价
- 折扣（%）
- 税率（%）
- 备注
- 操作（删除）

#### 数据流：
1. **新增套餐**：
   - 填写基本信息
   - 添加产品到 items 数组
   - 调用 POST /api/packages，将套餐和 items 一起发送
   - 后端自动保存套餐和所有产品关联

2. **编辑套餐**：
   - 调用 GET /api/packages/{id} 加载套餐（包含items）
   - 修改基本信息和产品列表
   - 调用 PUT /api/packages/{id} 保存
   - 后端先删除旧的关联，再保存新的关联

3. **删除套餐**：
   - 后端自动删除所有关联的产品记录

## 从套餐创建报价单

### QuotationServiceImpl.createFromPackage()
1. 验证套餐和客户存在
2. 创建报价单基本信息
3. 从套餐加载产品明细（`package_items` 表）
4. 将套餐产品转换为报价明细：
   - 只加载启用状态的产品
   - 价格优先级：套餐项价格 > 产品默认价格
   - 税率优先级：套餐项税率 > 产品税率 > 系统默认税率
5. 自动计算报价单总价

### 前端 QuotationForm.vue
- 选择套餐后，调用 `/api/quotations/from-package` 接口
- 后端创建报价单并返回完整数据（包含产品明细）
- 前端直接使用返回的数据填充表单

## 数据验证

1. **产品必填**：在表单中添加产品时，必须选择产品
2. **数量验证**：数量必须大于0
3. **价格验证**：单价不能为负
4. **税率验证**：税率范围 0-100%

## 测试建议

1. **新增套餐并添加产品**：
   - 创建新套餐
   - 添加多个产品
   - 保存后检查数据库是否正确保存

2. **编辑套餐产品**：
   - 编辑现有套餐
   - 修改产品列表
   - 保存后检查产品是否更新

3. **从套餐创建报价单**：
   - 选择套餐
   - 检查报价单是否正确加载了所有产品

4. **删除套餐**：
   - 删除套餐
   - 检查关联的产品记录是否被删除

## 注意事项

1. 套餐保存时，产品列表会完全替换（删除旧的，插入新的）
2. 编辑套餐时，如果清空了产品列表，所有关联会被删除
3. 从套餐创建报价单时，只加载启用状态的产品
4. 价格和税率的优先级规则确保报价单的灵活性
